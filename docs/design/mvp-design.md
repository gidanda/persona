# Persona MVP 設計ドキュメント

## 概要

Persona は「人を知るためのデジタル名刺」をコンセプトとしたサービス。
QRコードでプロフィールを交換し、SNSリンクや文化的な好み（本・映画・音楽など）、
現在考えていること・取り組んでいることを通して相手の人物像を短時間で知ることができる。

SNSのような「つながり」よりも **人を知ること** を重視する。

------------------------------------------------------------------------

# 画面設計

## 1. ログイン画面

目的: アカウント作成 / ログイン

機能 - Googleログイン - メールログイン

------------------------------------------------------------------------

## 2. 初期プロフィール作成

入力項目

-   実名
-   表示名
-   アバター画像
-   写真（1枚）
-   Bio（短文）

SNSリンク - X - Instagram - GitHub - その他

Taste（3つまで） - ジャンル選択 - 作品名入力

Thinking - 今考えていること（短文）

Doing - 今取り組んでいること（短文）

自由記入欄 - タイトル - 内容

------------------------------------------------------------------------

## 3. 自分のプロフィール画面

表示内容

-   avatar
-   名前
-   bio

Links - SNSリンク一覧

Taste - Book - Movie - Song 等

Thinking - 今考えていること

Doing - 今取り組んでいること

Free block - 自由記入

操作

-   QRコード表示
-   プロフィール編集
-   People一覧

------------------------------------------------------------------------

## 4. QRコード画面

用途: 名刺交換

表示内容

-   自分のQRコード
-   「Scan to view profile」

------------------------------------------------------------------------

## 5. 相手プロフィール画面

QR読み取り後に表示

表示

-   名前
-   bio
-   SNSリンク
-   Taste
-   Thinking
-   Doing

交換はQR読み取り時点で自動登録

Followボタンあり

------------------------------------------------------------------------

## 6. People画面

交換したユーザー一覧

構造

Following - よく見る人

Exchanged - 交換した人

------------------------------------------------------------------------

# データベース設計

## users

ユーザー基本情報

    id
    email
    username
    real_name
    display_name
    avatar_url
    profile_photo_url
    bio
    thinking_now
    doing_now
    free_block_title
    free_block_body
    visibility
    created_at
    updated_at

------------------------------------------------------------------------

## social_links

SNSリンク

    id
    user_id
    platform
    url
    display_order
    created_at

------------------------------------------------------------------------

## taste_entries

文化的好み

    id
    user_id
    category
    work_title
    creator_name
    display_order
    created_at
    updated_at

category 例

-   book
-   movie
-   song
-   game
-   anime
-   manga

------------------------------------------------------------------------

## exchanges

QR交換記録

    id
    from_user_id
    to_user_id
    created_at

QRを読み取った時点で作成される

------------------------------------------------------------------------

## follows

気になるユーザー

    id
    user_id
    target_user_id
    created_at

exchangeとは別に管理

------------------------------------------------------------------------

# データ関係

User

-   social_links
-   taste_entries
-   exchanges
-   follows

------------------------------------------------------------------------

# MVPに含める機能

-   ログイン（Google / Email）
-   プロフィール作成
-   SNSリンク登録
-   Taste登録（最大3）
-   Thinking / Doing
-   自由記入欄
-   QR交換
-   交換済み一覧
-   Follow

------------------------------------------------------------------------

# MVPに含めない機能

-   メッセージ
-   いいね
-   コメント
-   タイムライン
-   レコメンド
-   ランキング
