# デジタルプロフィールアプリ 画面仕様書（MVP）

## 1. 概要

### 1-1. アプリの目的
フィジカルな場で出会った相手と、QRコードを通じてデジタルプロフィールを交換・蓄積するためのアプリを提供する。

本アプリはSNSのようにオンライン上で自由に人とつながることを目的とせず、実際に会った相手との接点を記録・再確認しやすくすることを目的とする。

### 1-2. MVPの基本思想
- フィジカルな出会いを起点に交換する
- 交換はQRコード読み取りによってのみ成立する
- 人を肩書きで判断させすぎない
- 複雑な公開範囲や承認フローは設けない
- メッセージ機能は一旦保留にする

---

## 2. MVPで提供する主要機能
- ユーザー登録
- ログイン
- 初回プロフィール作成
- 自分のプロフィール表示・編集
- 自分のQRコード表示
- QRコード読み取り
- 交換済み相手の一覧表示
- 相手プロフィール詳細表示
- お気に入り登録
- 自動アバター表示

---

## 3. 交換に関する基本仕様

### 3-1. 交換成立条件
- どちらか一方が相手のQRコードを読み取った時点で交換成立とする
- 承認・非承認の概念は設けない

### 3-2. 交換の成立方法
- 交換はQRコード読み取りによってのみ成立する
- URLを別経路で受け取って開いても、交換成立とはみなさない
- URLはプロフィール参照用の識別子として使用する

### 3-3. 重複交換
- 既に交換済みの相手との重複作成は行わない
- 重複読み取り時は新規レコードを作成しない

### 3-4. 公開範囲設定
- MVPでは公開範囲設定機能は設けない
- 交換済み相手には同一プロフィールを表示する

---

## 4. プロフィール仕様

### 4-1. 登録時の必須項目
- ユーザーID
- パスワード
- 認証用情報
- 表示名
- 本名

### 4-2. プロフィール項目
#### 必須
- ユーザーID
- 表示名
- 本名

#### 任意
- 自己紹介
- SNSリンク
- プロフィール画像

### 4-3. プロフィール思想
- 肩書き・所属欄は設けない
- 人を肩書きでラベル化しすぎない設計を優先する
- ただし識別補助のため、短めの自己紹介欄は設ける

---

## 5. アバター仕様

### 5-1. 基本方針
- MVPではクラシック8bitピクセルキャラ型の自動アバターを採用する
- ユーザーIDをseedとして自動生成する
- 各ユーザーに固有の見た目を割り当てる

### 5-2. アバター表示ルール
- 初期状態では自動生成アバターを表示する
- 将来的なデザイン変更は可能な構造にする
- 必要に応じて後から画像アップロードに差し替えられる設計が望ましい

### 5-3. 採用理由
- 識別しやすい
- テック寄りの世界観に合う
- 軽量で一覧表示に向いている
- MVPとして実装しやすい

---

## 6. QR仕様

### 6-1. QRの中身
QRコードには以下の形式のURLを含める。

`https://persona.app/u/{user_id}`

例：
`https://persona.app/u/gouda`

### 6-2. URLの扱い
- URLはプロフィール識別のために使用する
- ただし交換成立はQR読み取り時のみとする
- 将来的にはWebプロフィール参照や共有用途への拡張が可能

## 8. 画面ごとの機能仕様

## 8-1. ログイン画面
### 目的
既存ユーザーがログインするための画面。

### 表示項目
- アプリ名 / ロゴ
- ユーザーID入力欄
- パスワード入力欄
- ログインボタン
- 新規登録画面への導線

### ユーザー操作
- ユーザーIDとパスワードを入力してログインする
- 新規登録画面へ移動する

### 成功時挙動
- 認証成功後、プロフィール作成済みならホーム画面へ遷移
- 未作成なら初回プロフィール作成画面へ遷移

### 失敗時挙動
- ユーザーIDまたはパスワード不一致エラーを表示
- 通信エラーを表示

---

## 8-2. 新規登録画面
### 目的
新規ユーザーがアカウントを作成するための画面。

### 表示項目
- アプリ名 / ロゴ
- ユーザーID入力欄
- パスワード入力欄
- パスワード確認欄
- 認証用情報入力欄
- 表示名入力欄
- 本名入力欄
- 登録ボタン
- ログイン画面への導線

### ユーザー操作
- 必須項目を入力してアカウントを作成する

### 成功時挙動
- 登録成功後、初回プロフィール作成画面へ遷移

### 失敗時挙動
- ユーザーID重複エラー
- パスワード不一致エラー
- 必須項目未入力エラー
- 認証失敗エラー
- 通信エラー

### 備考
- ユーザーIDはユーザー自身が決定する

---

## 8-3. 初回プロフィール作成画面
### 目的
交換に必要な最低限のプロフィール情報を整えるための画面。

### 表示項目
- 自動生成アバタープレビュー
- 自己紹介入力欄
- SNSリンク入力欄
- 保存ボタン
- スキップまたは後で設定の導線（任意）

### ユーザー操作
- 任意項目を入力して保存する
- 必要最小限のまま先へ進む

### 成功時挙動
- 保存後、ホーム画面へ遷移

### 失敗時挙動
- URL形式不正エラー
- 保存失敗エラー

### 備考
- 表示名・本名などの必須情報は登録時に取得済み
- 画像アップロードを入れる場合はこの画面または編集画面で対応する

---

## 8-4. ホーム画面
### 目的
アプリ利用開始後の起点となる画面。

### 表示項目
- ヘッダー
- 自分のアバター
- 自分のQRを表示ボタン
- QRを読み取るボタン
- 最近交換した相手の簡易一覧（任意）
- 下部ナビゲーション

### ユーザー操作
- 自分のQR表示画面へ進む
- QR読み取り画面へ進む
- つながり一覧へ進む
- 自分のプロフィールへ進む

### 成功時挙動
- 各ボタン押下で対応画面へ遷移

### 備考
- 交換人数などのサマリー表示は設けない

---

## 8-5. 自分のQR表示画面
### 目的
相手に自分のQRコードを見せて交換してもらうための画面。

### 表示項目
- 自分のアバター
- 表示名
- 本名
- 自分専用QRコード
- 補助テキスト
- 戻るボタン

### ユーザー操作
- 相手にQRコードを提示する

### 成功時挙動
- 特別な更新は不要

### 失敗時挙動
- QR生成失敗エラー
- 読み込み失敗エラー

---

## 8-6. QR読み取り画面
### 目的
相手のQRコードを読み取り、交換処理を行うための画面。

### 表示項目
- カメラプレビュー
- 読み取りガイドUI
- カメラ権限案内
- キャンセルボタン

### ユーザー操作
- 相手のQRコードをスキャンする
- 読み取りをキャンセルする

### 成功時挙動
- QR読み取り成功時、交換処理を実行
- 処理成功後、読み取り成功画面へ遷移

### 失敗時挙動
- カメラ権限なし
- 無効なQRコード
- 自分自身のQRを読み取った場合のエラー
- 相手ユーザーが存在しない場合のエラー
- 通信失敗
- 既に交換済みの相手を読み取った場合の案内表示

---

## 8-7. 読み取り成功画面
### 目的
交換成功を短く明確に伝えるための画面。

### 表示項目
- 交換成功メッセージ
- 相手のアバター
- 相手の表示名
- 相手の本名
- 戻る案内またはカウントダウン表現

### ユーザー操作
- 画面をタッチする
- 一定時間待つ

### 成功時挙動
- 一定時間経過またはタッチ操作でQR読み取り画面へ戻る
- 連続して複数人を読み取れる導線にする

### 備考
- 読み取り画面自体をキャンセルしたときに元画面へ戻る構成とする

---

## 8-8. つながり一覧画面
### 目的
交換済み相手を一覧で確認するための画面。

### 表示項目
- 検索欄
- フィルタタブ
  - すべて
  - お気に入り
- つながりカード一覧
  - アバター
  - 表示名
  - 本名
  - 自己紹介の一部または補助情報
  - お気に入り状態
- 空状態表示

### ユーザー操作
- 相手を検索する
- お気に入りのみ表示する
- 一覧から相手詳細へ進む
- お気に入りON/OFFを切り替える

### 成功時挙動
- フィルタや検索条件に応じて一覧更新
- お気に入り状態を更新

### 失敗時挙動
- 一覧取得失敗
- お気に入り更新失敗

### 備考
- 並び順は新しい交換順を基本とする

---

## 8-9. 相手プロフィール詳細画面

### 表示項目

- SNSリンク

### ユーザー操作
- 相手プロフィールを閲覧する
- SNSリンクを開く
- お気に入り状態を切り替える

### 成功時挙動
- お気に入り状態を更新

### 失敗時挙動
- 詳細取得失敗
- 更新失敗

### 備考
- フォロー機能は設けない

---

## 8-10. 自分のプロフィール画面

- 編集ボタン
- 自分のQR表示への導線

## 9. ユーザーフロー

### 9-1. 初回利用フロー
1. 新規登録
2. 初回プロフィール作成
3. ホーム画面へ遷移

### 9-2. 通常利用フロー
1. ログイン
2. ホーム画面
3. QR表示またはQR読み取り
4. つながり一覧確認
5. 相手プロフィール詳細確認

### 9-3. 交換フロー
1. ホーム画面からQR読み取り画面へ遷移
2. 相手のQRコードを読み取る
3. 交換処理実行
4. 読み取り成功画面を表示
5. 一定時間経過またはタッチで読み取り画面へ戻る
6. キャンセル操作で元画面へ戻る

---

## 10. MVPで除外・保留する機能
- フォロー機能
- 公開範囲設定
- 肩書き / 所属欄
- ホーム画面の交換人数サマリー
- 承認制交換
- メッセージ機能（保留）
- ブロック / 通報
- 高度な検索・高度なフィルタ

---

## 11. 実装上の重要ルール
- 交換成立はQR読み取り時のみ
- URLアクセスだけでは交換成立しない
- 交換の重複作成はしない
- お気に入りのみ実装し、フォローは実装しない
- 本名は必須項目とする
- ユーザーIDはユーザーが自分で決定する
- アバターはクラシック8bitピクセルキャラを採用する

---

## 12. API設計（MVP）

本章では、フロントエンドとバックエンドの通信仕様として使用するAPIを定義する。

### 12-1. APIカテゴリ
MVPではAPIを以下の5カテゴリに分けて設計する。

- 認証
- 自分のプロフィール
- QR交換
- つながり
- お気に入り

---

## 13. 認証API

### 13-1. 新規登録
`POST /api/auth/signup`

#### リクエスト
```
{
  "userId": "gouda",
  "password": "password123",
  "displayName": "Yuki",
  "realName": "裕樹 合田",
  "verification": "example@example.com"
}
```

#### レスポンス
```
{
  "user": {
    "id": "usr_123",
    "userId": "gouda",
    "displayName": "Yuki",
    "realName": "裕樹 合田"
  },
  "accessToken": "token"
}
```

#### バリデーション
- userId重複不可
- password最小文字数
- displayName必須
- realName必須

---

### 13-2. ログイン
`POST /api/auth/login`

#### リクエスト
```
{
  "userId": "gouda",
  "password": "password123"
}
```

#### レスポンス
```
{
  "user": {
    "id": "usr_123",
    "userId": "gouda",
    "displayName": "Yuki",
    "realName": "裕樹 合田",
    "profileCompleted": true
  },
  "accessToken": "token"
}
```

---

### 13-3. 自分情報取得
`GET /api/auth/me`

#### レスポンス
```
{
  "id": "usr_123",
  "userId": "gouda",
  "displayName": "Yuki",
  "realName": "裕樹 合田",
  "profileCompleted": true
}
```

---

## 14. 自分のプロフィールAPI

### 14-1. 自分のプロフィール取得
`GET /api/me/profile`

#### レスポンス
```
{
  "userId": "gouda",
  "displayName": "Yuki",
  "realName": "裕樹 合田",
  "bio": "AIやロボティクスに興味があります。",
  "avatarUrl": "https://persona.app/avatar/gouda",
  "snsLinks": []
}
```

---

### 14-2. プロフィール更新
`PUT /api/me/profile`

#### リクエスト
```
{
  "bio": "AIやロボティクスに興味があります。",
  "snsLinks": [
    {
      "type": "x",
      "url": "https://x.com/example"
    }
  ]
}
```

---

## 15. QR関連API

### 15-1. 自分のQR情報取得
`GET /api/me/qr`

#### レスポンス
```
{
  "qrValue": "https://persona.app/u/gouda"
}
```

---

### 15-2. QR読み取りによる交換作成
`POST /api/exchanges`

#### リクエスト
```
{
  "qrValue": "https://persona.app/u/tanaka"
}
```

#### レスポンス（新規交換）
```
{
  "status": "created",
  "connection": {
    "id": "con_001",
    "partner": {
      "userId": "tanaka",
      "displayName": "Taro",
      "realName": "田中 太郎",
      "avatarUrl": "https://persona.app/avatar/tanaka"
    },
    "exchangedAt": "2026-03-10T12:00:00Z"
  }
}
```

#### レスポンス（既に交換済み）
```
{
  "status": "already_exists",
  "connection": {
    "id": "con_001"
  }
}
```

## 12. 補足
本仕様はMVPを対象としており、まずは「出会った人を交換・記録・見返す」最小価値の実現を優先する。将来的には以下の拡張余地がある。

- メッセージ機能
- アバターデザイン変更
- 画像アップロード強化
- 交換履歴の高度化
- Webプロフィール共有導線の追加
- 名刺交換・イベント特化機能
