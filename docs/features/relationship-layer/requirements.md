# Persona Relationship Layer 要件定義書

作成日: 2026-03-17

参照元:

- [`overview.md`](/Users/goudayuuki/dev/persona/docs/features/relationship-layer/overview.md)

## 1. 文書の目的

本書は、Persona に追加する `Relationship Layer` の機能要件を定義する。

対象は、既存 MVP のプロフィール交換体験の上に追加する以下の 3 機能である。

- `Reaction`
- `Shared`
- `Thread`

## 2. 背景

既存の Persona MVP は、プロフィールを作る、交換する、見る、までの導線を提供している。

一方で、交換後に関係を前進させる導線は弱く、接点が一度きりで終わりやすい。Relationship Layer では、プロフィール上の興味対象に対する軽い反応を起点に、2 人の間に共有話題を生み、必要な場合だけ小さな会話を始められる状態を目指す。

## 3. 目的

Relationship Layer の目的は次の通り。

- 会った後の関係継続を支援する
- 相手の興味やカルチャーへの反応を自然に起こせるようにする
- 2 人の間に Shared Topic を蓄積し、関係の履歴として見返せるようにする
- 汎用チャット化せず、話題単位の会話に限定する

## 4. 対象ユーザー

- 既に QR 交換を完了した相手を持つユーザー
- 相手のプロフィールを見返しながら、共通話題や興味対象をきっかけに関係を深めたいユーザー
- SNS のような常時会話ではなく、必要なときだけ軽く反応したいユーザー

## 5. スコープ

### 5.1 MVP に含めるもの

- Bottom Navigation への `Shared` 追加
- 他者プロフィール項目への `Reaction`
- `Reaction` を起点とした `Shared Topic` 自動生成
- `Shared` 一覧画面
- 相手ごとの `Shared Topic` 一覧
- `Shared Topic` 詳細画面
- `Thread` の開始、一覧、送信
- `Thread` を最初のテキスト送信で開始する挙動
- 同一の 2 人と同一対象に対する Shared Topic の統合

### 5.2 MVP に含めないもの

- Home フィード
- Together / Experienced
- アルバム
- AI による推薦や共通点抽出
- 高度な外部コンテンツ API 連携
- 汎用 DM
- 既読、通知、リアクション数競争のような SNS 機能

## 6. ユースケース

### 6.1 プロフィール閲覧から共有話題が生まれる

1. ユーザーが People から交換済み相手のプロフィールを見る
2. 音楽、本、映画、場所、興味テーマなどの項目に反応する
3. システムが 2 人の関係の中に Shared Topic を作成または既存統合する
4. Shared 画面からその話題を見返せる

### 6.2 Shared Topic から会話を始める

1. ユーザーが Shared Topic 詳細を開く
2. まだ会話が始まっていない場合は、入力欄から最初のメッセージを送る
3. 送信時点で Thread が開始される
4. 以後はその Shared Topic に紐づく会話として表示される

### 6.3 同じ話題への再反応

1. 同じ 2 人の間で同じ対象に再度反応や推薦が行われる
2. システムは新規 Shared Topic を乱立させず既存 Topic に統合する
3. 最終更新日時や発生理由の履歴が更新される

## 7. 機能要件

### 7.1 Reaction 要件

- 交換済み相手のプロフィール項目に対して反応できること
- Reaction は軽量なアクションであること
- MVP では少数の固定種別で開始すること
- Reaction 実行時に Shared Topic の生成判定が走ること
- 自分自身のプロフィールには Reaction できないこと

### 7.2 Shared 要件

- Shared 画面から、話題が生まれている相手一覧を見られること
- 相手単位で Shared Topic 一覧を見られること
- Shared Topic ごとに対象、発生理由、会話有無、最終更新を確認できること
- 同一話題は統合されること
- 会話が未開始の Topic も一覧に保持されること

### 7.3 Thread 要件

- Shared Topic ごとに独立した会話を持てること
- Shared Topic があるだけでは Thread が開始されないこと
- 最初のテキスト送信時に Thread が開始されること
- Topic と無関係な常設チャットへ拡張しないこと

### 7.4 導線要件

- Bottom Navigation に `Shared` を追加すること
- 他者プロフィールから Shared へ自然に遷移できること
- People と Shared の間で、相手起点と話題起点の両方の見方ができること

## 8. 非機能要件

### 8.1 UX 要件

- 反応は数タップで完了すること
- 空のチャットを大量に発生させないこと
- 一覧が SNS タイムラインのように騒がしくなりすぎないこと
- 情報量は必要最小限に保ち、相手との関係が把握しやすいこと

### 8.2 設計要件

- Relationship Layer は既存の `People / Scan / Me` 構造を壊さず追加できること
- データ構造は将来的な `Together / Experienced` や外部コンテンツ連携に拡張可能であること
- Shared と Thread を明確に分離したモデルであること

## 9. 成功条件

以下を満たせば MVP として成立とみなす。

- プロフィール項目への反応から Shared Topic が生まれる
- Shared 画面で相手ごとの共有話題を見返せる
- Shared Topic ごとに必要なときだけ会話を始められる
- 同じ話題が乱立せず、2 人の関係履歴としてまとまって見える

## 10. 今後の拡張前提

本要件定義は MVP 用であり、後続では以下を拡張対象とする。

- Home 導線
- Together / Experienced
- Recommendation 強化
- 外部コンテンツ ID との厳密な紐づけ
- AI 補助
